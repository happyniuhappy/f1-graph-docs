# 概述

Neo4j-OGM 用于各种各样的生态系统，但最特别的是在更广泛的 Spring 生态系统中。Neo4j-OGM 主要为Spring Data Neo4j (SDN) 做工作，所以 SDN 依赖它。同时，SDN也依赖于Spring Framework。Spring Boot 依赖于 Spring Framework 和 Spring Data Commons 以及 SDN。

这为混合不能很好地协同工作或根本不能正常工作的版本提供了很大的潜力。因此，我们为 Bolt（又名服务器）传输模式推荐的组合概述如下：

# 使用Spring Boot starter

图数据库提供neo4j-java-driver-spring-boot-starter。这个 Spring Boot 模块提供了 Java / Bolt 驱动程序的所有属性，并将驱动程序的实例显式公开为 bean，而无需在您的应用程序上强制执行 OGM。然而，它与 OGM 兼容。

您可以组合以下版本：

neo4j-java-driver-spring-boot-starter Spring Boot Neo4j Java Driver Neo4j-OGM

重要提示：如果您计划通过 Bolt 将 Neo4j-OGM 与即将推出的 Neo4j 4.3 数据库一起使用并计划继续使用我们的 AutoIndexManager，请确保您使用的是 Neo4j-OGM >= 3.2.24。

# Neo4j-OGM嵌入案例

重要提示：启动从 OGM 嵌入的 Neo4j 并使用嵌入的 API 仅支持并包括 Neo4j 3.5。如果您的应用程序需要嵌入式 4.0 数据库，请将其创建为标准 bean 并使用我们的 Neo4j Java 驱动程序打开本地螺栓套接字连接。

Neo4j-OGM 提供嵌入模式并为您启动嵌入实例。这在开发和部署过程中都很方便。但请记住，您将自己与数据库版本紧密结合，不仅与您的对象图映射框架，而且与您的应用程序框架密切相关。例如，如果您启动了基于 OGM 嵌入式 2.1.x 和数据库 3.1 的 Spring Boot 应用程序，则不能简单地将数据库依赖项或 OGM 依赖项交换到高于数据库 3.1 的内容，因为上述组件是相互依赖的。

如果您真的想在生产中运行嵌入式数据库，您应该考虑自己提供数据库 bean 并显式打开嵌入式数据库螺栓端口并使用 OGM 螺栓传输。
